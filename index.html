Instagram Clone

Insta485 is a set of 3 projects completed in UM's EECS 485: Web Systems course, focusing on full-stack development to create an Instagram-like platform.

The projects aimed to replicate the basic user functionalities of an image-sharing site, involving:

  • A frontend developed with React, Jinja, HTML, and CSS, and served through Python's Flask framework.
  • A backend REST API constructed using Python's Flask and a SQLite database.
  • Deployment to AWS utilizing services like EC2.

EECS 485 stands out as one of my favorite courses at UM, largely due to projects like these. It provided me with invaluable experience in collaborative team work on a significant software project. I was able to enhance my full-stack development skills and fill gaps in my knowledge, including tasks like writing bash scripts to automate workflows. This class proved to be pivotal in preparing me for a successful career in Software Engineering.

DOGU Internship

During my Software Engineer Internship at DOGU CO., LTD, I focused on enhancing indoor robotics capabilities, particularly in overcoming obstacles and maintaining accurate location awareness.

  • One of my responsibilities involved conducting a comprehensive market analysis to gain insights into current trends, challenges, and opportunities for indoor robotics in healthcare settings. This involved researching healthcare professionals and patients to understand their specific needs and challenges in utilizing indoor robots for various tasks within healthcare facilities.
  • I actively contributed to the development of Vector Maps on top of Point Cloud Data (PCD) maps using Vector Map Builder (Autoware Map Tools). This task required a deep understanding of mapping technologies and their application in indoor robotic navigation.
  • I had the chance to test advanced sensor technology, including cameras and sensors, to detect potential dangers and ensure swift responses in hospital environments. This aspect of my internship provided valuable hands-on experience in working with cutting-edge technologies crucial for the development of safe and efficient indoor robotic systems.

    Stock Simulation

    The program received as input a list of orders that specified whether the trader intended to buy or sell a specific stock. As output, the program would simulate a stock market by printing when a certain stock is bought or sold when it is profitable to do so. I also implemented an algorithm to calculate the ideal time that a time traveler theoretically could have bought shares of a stock and then later sold that stock to maximize profit. Completing this project gave me experience using priority queues and the STL and it helped me become more proficient with object-oriented design.
